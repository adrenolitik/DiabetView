## 📋 Варианты развертывания

### Вариант 1: Через веб-интерфейс Vercel (Рекомендуется)

Это самый простой способ для развертывания!

1. **Перейдите на Vercel**
   - Откройте [vercel.com](https://vercel.com)
   - Войдите через GitHub аккаунт

2. **Импортируйте проект**
   - Нажмите "Add New..." → "Project"
   - Выберите репозиторий `DiabetView` из списка
   - Нажмите "Import"

3. **Настройка проекта**
   - Vercel автоматически определит настройки Vite
   - Framework Preset: `Vite`
   - Build Command: `npm run build`
   - Output Directory: `dist`

4. **Переменные окружения (Опционально)**
   - Если хотите использовать настоящий Google Gemini API:
     - Раздел "Environment Variables"
     - Добавьте: `VITE_GOOGLE_API_KEY` = ваш_ключ
   - Без API ключа приложение работает с mock-данными

5. **Разверните**
   - Нажмите "Deploy"
   - Ожидайте завершения (обычно 1-2 минуты)
   - Получите ссылку вида: `https://diabetview.vercel.app`

### Вариант 2: Через Vercel CLI

Если вы предпочитаете командную строку:

1. **Войдите в Vercel**
   ```bash
   npx vercel login
   ```
   Следуйте инструкциям для входа через браузер.

2. **Разверните проект (Preview)**
   ```bash
   cd /path/to/DiabetView
   npx vercel
   ```
   - Следуйте интерактивным подсказкам
   - Выберите настройки по умолчанию
   - Получите preview URL

3. **Production развертывание**
   ```bash
   npx vercel --prod
   ```
   Это создаст production версию на постоянном URL.

## ⚙️ Настройка переменных окружения

### В Vercel Dashboard:

1. Откройте настройки проекта
2. Перейдите в "Environment Variables"
3. Добавьте переменные:

| Имя переменной | Значение | Описание |
|---|---|---|
| `VITE_GOOGLE_API_KEY` | ваш_api_ключ | Google Gemini API key (опционально) |

### Локально (для тестирования):

Создайте файл `.env.local`:
```env
VITE_GOOGLE_API_KEY=your_actual_api_key_here
```

## 🐛 Возможные проблемы и решения

### Build Failed
**Проблема**: Ошибка при сборке на Vercel

**Решение**: 
```bash
# Проверьте локально
npm run build

# Если ошибка, проверьте:
# - node_modules установлены
# - нет TypeScript ошибок
```

### API Key не работает
**Проблема**: Gemini API не отвечает

**Решение**:
- Приложение автоматически переключится на mock-данные
- Проверьте формат ключа в переменных окружения
- Убедитесь, что ключ имеет право на Gemini API

### 404 на страницах
**Проблема**: Прямые ссылки возвращают 404

**Решение**:
- `vercel.json` уже настроен с правильными rewrites
- Vercel автоматически применит конфигурацию

## 📊 Технические характеристики

- **Framework**: Vite + React 18
- **Язык**: TypeScript
- **Стили**: Tailwind CSS
- **Графики**: Recharts
- **AI**: Google Gemini API (с fallback на mock)
- **Размер сборки**: ~590 KB (gzipped: ~168 KB)
- **Время сборки**: ~6-10 секунд
